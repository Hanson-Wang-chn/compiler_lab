# 注释处理测试

## 功能说明

本测试程序专门测试词法分析器对C语言两种注释格式的处理能力。

## 测试内容

### 1. 块注释测试
- 单行块注释：`/* 注释 */`
- 多行块注释：
  ```c
  /* 这是
     多行注释 */
  ```
- 空块注释：`/**/`

### 2. 行注释测试
- 标准行注释：`// 注释内容`
- 行尾注释：`int x = 10; // 变量定义`
- 连续行注释：
  ```c
  // 注释1
  // 注释2
  ```

### 3. 特殊情况测试
- 注释中的特殊字符：`/* !@#$%^&*() */`
- 注释紧邻代码：`int/*注释*/x;`
- 字符串中的注释符号：`"// 这不是注释"`

### 4. 边界测试
- 未闭合块注释（文件末尾）
- 空文件
- 只有注释的文件

## 编译和运行

```bash
cd comment_test
g++ -std=c++11 -o comment_test comment_test.cpp
./comment_test
```

## 验证要点

1. **映射正确性**：
   - 块注释 `/* */` 映射到编号79
   - 行注释 `//` 映射到编号79
   - 输出格式：`</* 注释内容 */,79>` 或 `<// 注释内容,79>`

2. **内容保留**：
   - 注释的实际文本内容会被完整保留
   - 块注释包含开始符`/*`和结束符`*/`
   - 行注释包含开始符`//`和后续内容

3. **识别准确性**：
   - 不误判字符串中的`//`或`/*`
   - 正确处理注释后的代码
   - 注释内的特殊字符正确保存

4. **鲁棒性**：
   - 处理未闭合注释
   - 处理空注释
   - 处理注释嵌套（外层识别为注释）

## 技术实现

1. **块注释识别**：检测`/*`，扫描并保存到`*/`为止
2. **行注释识别**：检测`//`，扫描并保存到行尾为止
3. **内容保留**：保留注释的完整文本内容
4. **编号统一**：两种注释都映射到编号79
